# MyMusic - 本地音乐播放器

一个功能完整的基于 Electron 的本地音乐播放器应用程序。

## 功能特性

### 🎵 核心功能
- **音乐库管理**: 导入和管理本地音乐文件，支持多种音频格式（MP3, WAV, FLAC, AAC, M4A, OGG, WMA）
- **智能播放器**: 播放/暂停、上一曲/下一曲、进度控制、音量调节
- **播放模式**: 支持顺序播放、随机播放、单曲循环、列表循环
- **播放列表管理**: 创建、编辑、删除自定义播放列表

### 🎨 用户界面
- **现代化设计**: 简洁美观的界面，支持深色/浅色主题
- **响应式布局**: 适配不同屏幕尺寸
- **直观导航**: 清晰的侧边栏导航和内容组织

### 🎛️ 高级功能
- **音乐分类**: 按艺术家、专辑、流派自动分类
- **收藏系统**: 收藏喜爱的歌曲和艺术家
- **搜索功能**: 智能搜索歌曲、艺术家、专辑
- **歌词显示**: 支持 LRC 格式歌词同步显示
- **音效设置**: 内置多种音效预设（流行、摇滚、古典、爵士等）

### ⚙️ 系统功能
- **数据持久化**: 本地存储播放列表、收藏、设置等
- **键盘快捷键**: 丰富的快捷键支持
- **文件完整性检查**: 自动检测和清理无效文件
- **数据导入导出**: 备份和恢复应用数据

## 技术架构

### 前端技术
- **HTML5**: 语义化结构
- **CSS3**: 现代化样式和动画
- **JavaScript ES6+**: 模块化开发
- **Electron**: 跨平台桌面应用框架

### 核心模块
1. **存储管理 (storage.js)**: localStorage 数据管理
2. **音乐库 (musicLibrary.js)**: 文件导入和音乐数据管理
3. **播放器 (player.js)**: 音频播放控制和状态管理
4. **播放列表 (playlist.js)**: 播放列表创建和管理
5. **用户界面 (ui.js)**: 界面交互和主题管理
6. **工具函数 (utils.js)**: 通用工具和辅助函数

## 安装和运行

### 前置要求
- Node.js (v14 或更高版本)
- npm 或 yarn

### 安装步骤
```bash
# 1. 安装 Node.js
# 访问 https://nodejs.org/ 下载并安装

# 2. 克隆项目
git clone <repository-url>
cd mymusic

# 3. 安装依赖
npm install

# 4. 启动应用
npm start

# 5. 开发模式（带开发工具）
npm run dev
```

## 使用说明

### 导入音乐
1. 点击顶部的"导入文件"或"导入文件夹"按钮
2. 选择音乐文件或包含音乐的文件夹
3. 应用会自动扫描并添加音乐到库中

### 播放控制
- **空格键**: 播放/暂停
- **左右箭头**: 上一曲/下一曲
- **上下箭头**: 音量控制
- **M键**: 静音切换
- **F键**: 收藏当前歌曲
- **S键**: 随机播放切换
- **R键**: 循环模式切换

### 创建播放列表
1. 在左侧导航栏点击"创建播放列表"
2. 输入播放列表名称
3. 右键点击歌曲选择"添加到播放列表"

### 主题和设置
1. 点击右上角的设置按钮
2. 在设置面板中调整主题、播放选项等

## 开发说明

### 添加新功能
1. 在相应的模块文件中添加功能代码
2. 在 HTML 中添加必要的界面元素
3. 在 CSS 中添加样式
4. 更新相关的事件绑定和数据处理

### 自定义主题
1. 在 `src/css/themes.css` 中定义新的 CSS 变量
2. 创建新的主题类（如 `.theme-custom`）
3. 在设置中添加主题选项

### 扩展音频格式支持
1. 在 `utils.js` 的 `isAudioFile` 函数中添加新格式
2. 测试新格式的播放兼容性

## 故障排除

### 常见问题
1. **音乐文件无法播放**: 检查文件格式是否受支持
2. **界面显示异常**: 尝试刷新应用或重启
3. **文件导入失败**: 确保文件路径正确且有读取权限

### 重置应用
如果遇到严重问题，可以在开发者控制台中执行：
```javascript
window.app.resetApp()
```
